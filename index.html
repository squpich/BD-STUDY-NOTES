База данных и СУБД

Что такое база данных

База данных — организованный набор связанных данных, который можно хранить, искать, изменять и удалять.

В реляционной базе данные обычно представлены таблицами. Таблица состоит из столбцов и строк. Столбец описывает характеристику, а строка — один объект.

Запомни

Столбец = характеристика. Строка = конкретная запись.

Что такое СУБД

СУБД — система управления базами данных. Она организует и управляет одной или несколькими базами данных.

Зачем это нужно

  • создавать базы и таблицы;
  • сохранять и изменять данные;
  • искать информацию;
  • управлять доступом;
  • поддерживать целостность и транзакции;
  • обеспечивать резервирование и доступность.

PRIMARY KEY и FOREIGN KEY

PRIMARY KEY однозначно определяет строку. FOREIGN KEY ссылается на ключ другой таблицы. 

CREATE TABLE customers (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(100)
);

CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    FOREIGN KEY (customer_id)
        REFERENCES customers(customer_id)
);

Как запомнить

PRIMARY — главный идентификатор. FOREIGN — ссылка в другую таблицу.

Элементы Transact-SQL

Связи таблиц и JOIN

JOIN и ON

JOIN объединяет связанные строки из нескольких таблиц. ON задаёт условие связи. 

SELECT b.buyer_name, s.buyer_id, s.qty
FROM buyers AS b
INNER JOIN sales AS s
    ON b.buyer_id = s.buyer_id;

Как объяснить преподавателю

JOIN связывает таблицы, обычно по PRIMARY KEY и FOREIGN KEY. ON указывает общие столбцы.

Агрегатные функции вычисляют одно итоговое значение по набору строк.

Агрегатные функции

SUM Сумма
AVG Среднее
MIN Минимум
MAX Максимум

Обычно готовят три сервера, конфигурируют инстанции и подключают Router.

  1. Три машины с разными hostname.
  2. MySQL Server и MySQL Shell.
  3. Административный пользователь.
  4. dba.configureInstance для каждого узла.
  5. dba.createCluster.
  6. cluster.addInstance.
  7. MySQL Router.
  8. Проверка failover.

MySQL Cluster / NDB Cluster

MySQL NDB Cluster — распределённая shared-nothing база для высокой доступности и OLTP.

Компоненты

  • Management Node — конфигурация и управление;
  • Data/Storage Node — хранение данных;
  • SQL/API Node — SQL и приложения;
  • Node Group — реплики фрагментов.

Плюсы

  • синхронная репликация;
  • автоматическое sharding;
  • масштабирование чтения и записи;
  • географическая репликация;
  • высокая доступность.

Минусы

  • сложная эксплуатация;
  • не лучший вариант для тяжёлого OLAP.

FAQ

Почему DELETE * неверно?

DELETE удаляет строки, поэтому список столбцов не указывается.

Почему WHERE и HAVING разные?

WHERE фильтрует строки, HAVING — группы.

MongoDB заменяет SQL?

Нет. Это другая модель для других задач.
