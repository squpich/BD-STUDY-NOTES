Secção 1

Base de dados e SGBD

O que é uma base de dados

Uma base de dados é um conjunto organizado de dados relacionados que pode ser armazenado, pesquisado, alterado e eliminado.

Numa base de dados relacional, os dados são normalmente representados em tabelas. Uma tabela é composta por colunas e linhas. A coluna descreve uma característica, e a linha representa um objeto.

Para memorizar

Coluna = característica. Linha = registo concreto.

O que é um SGBD

O SGBD é o sistema de gestão de bases de dados. Organiza e gere uma ou várias bases de dados.

Para que serve

  • criar bases de dados e tabelas;
  • guardar e alterar dados;
  • pesquisar informação;
  • gerir o acesso;
  • garantir a integridade e as transações;
  • assegurar a redundância e a disponibilidade.

No exame

Resposta: os SGBD foram criados para organizar e gerir uma ou várias bases de dados.

Modelo relacional, OLTP e OLAP

Uma base de dados relacional guarda os dados em tabelas, e as relações são construídas através de chaves.

OLTP
Muitas operações diárias curtas: compra, reserva, transferência.
OLAP
Consultas analíticas, relatórios e estatísticas sobre grandes volumes de dados.

Qual é a diferença

O OLTP processa transações. O OLAP analisa os dados acumulados.

Secção 2

SQL e Transact-SQL

O que é o SQL

O SQL é a linguagem para definir a estrutura da base de dados, gerir dados e obter informação.

O Transact-SQL, ou T-SQL, é a implementação do SQL no Microsoft SQL Server. Baseia-se no SQL-92 e acrescenta variáveis, condições, ciclos, funções e procedimentos.

Muito resumidamente

O SQL é a linguagem geral. O T-SQL é a variante do SQL para o SQL Server.

PRIMARY KEY e FOREIGN KEY

A PRIMARY KEY identifica de forma única a linha. A FOREIGN KEY faz referência à chave de outra tabela. 

CREATE TABLE customers (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(100)
);

CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    FOREIGN KEY (customer_id)
        REFERENCES customers(customer_id)
);

Como memorizar

PRIMARY é o identificador principal. FOREIGN é a referência para outra tabela.

JOIN e ON

O JOIN combina linhas relacionadas de várias tabelas. O ON define a condição de ligação. 

SELECT b.buyer_name, s.buyer_id, s.qty
FROM buyers AS b
INNER JOIN sales AS s
    ON b.buyer_id = s.buyer_id;

O JOIN liga tabelas, normalmente através da PRIMARY KEY e da FOREIGN KEY. O ON indica as colunas comuns.

As funções agregadas calculam um único valor final a partir de um conjunto de linhas.

Funções agregadas
COUNT Número de valores
COUNT(*) Número de linhas

Um trigger é executado automaticamente perante um evento na tabela.

  • INSERT
  • UPDATE

Replicação

A replicação cria e atualiza cópias dos dados em vários servidores.

Modos de replicação
Assíncrono Rápido, mas pode haver perda das últimas alterações se o master falhar.
Semissíncrono O master espera a confirmação de receção da alteração pela réplica.
Virtualmente síncrono Group Replication no InnoDB Cluster.
Síncrono O NDB Cluster confirma a aplicação nos nós.

MySQL Cluster / NDB Cluster

O MySQL NDB Cluster é uma base de dados distribuída shared-nothing para alta disponibilidade e OLTP.

Componentes

  • Management Node — configuração e gestão;
  • Data/Storage Node — armazenamento de dados;
  • SQL/API Node — SQL e aplicações;
  • Node Group — réplicas dos fragmentos.

Vantagens

  • replicação síncrona;
  • sharding automático;
  • escalabilidade de leitura e escrita;
  • replicação geográfica;
  • alta disponibilidade.

Desvantagens

  • operação complexa;
  • não é a melhor opção para OLAP pesado.

6. Como instalar o MySQL no Linux e no Windows?

9. O que faz o * em SQL?

No SELECT seleciona todas as colunas; no COUNT(*) conta todas as linhas.

FAQ

Porque é que DELETE * está errado?

O DELETE elimina linhas, por isso não se indica uma lista de colunas.

Porque é que o WHERE e o HAVING são diferentes?

O WHERE filtra linhas, o HAVING filtra grupos.

O MongoDB substitui o SQL?

Não. É um modelo diferente para problemas diferentes.
